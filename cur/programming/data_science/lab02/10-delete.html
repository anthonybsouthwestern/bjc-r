The DELETE column and DELETE row blocks are used to remove data from a Table. They are all command blocks, so their changes to the Table are permanent. We can delete a column with either its index or its label, but we can only delete rows by their index because they do not have labels. Additionally, we can delete several columns at once using a list of column names.

The DELETE blocks each take in 2 arguments. These arguments are:

  1. A column label, index, or list of labels; A list of column indexes; A row index
  2. A Table

To see an example of the DELETE blocks in use, we have an example query for each of the 4 DELETE block variants. Do not run any of these in the Snap! file, but observe how they would be called if you needed to call them.

  1. The first call deletes the tolls column from the Table
  2. The second call deletes the 7th column from the Table, which in this case is the tolls column
  3. The third call simultaneously deletes the pickup_hour and dropoff_hour columns
  4. The fourth call simultaneously deletes the 15th and 16th columns, which in this case are the pickup_hour and dropoff_hour columns
  5. The fifth call deletes the third row from the Table

Be very cautious and precise with the way you delete data from a Table. These actions are very difficult to undo, so before you use a DELETE block, make sure you know for sure that you want to delete that data.
