Once we query a Table using GROUP or PIVOT, we can answer more specific questions about the data by accessing the Table we get. In the case of a GROUP call, the returned value is a Table, so we can access it using DASIS blocks we have seen before. In the case of a PIVOT call, the result is not a Table, so we must access it like a Snap! list with the block.

Using this strategy, use the GROUP and PIVOT blocks on the cleaned version of your movies Table to answer the following questions. You can make this helper block and use it as necessary.

  1. Who has directed the most movies in this dataset?
  2. On average, which decade in the dataset had the longest movies? (Hint: use GROUP's optional grouping function argument)
  3. How many comedy movies were released in the 1960s? (Hint: add a new column to the Table that indicates the decade in which a given movie was released)
  4. How many science fiction movies were released in the 1980s?
