The SELECT block is a block we can use to get a new Table containing only a small number of columns from the original Table. If you only need a few columns from a Table to do additional work, the SELECT block can eliminate columns you don't need.

The Table that results from a SELECT block call has at most the same number of columns (if you selected every column), but typically fewer columns than the original table, and it will also have the same number of rows as the original Table. The order of the columns in the new Table will be the same as their order in the original Table, just with the unselected columns omitted.

The SELECT block takes in 2 arguments. These arguments are:

  1. A list of column names or indexes to select (you can include both, but we recommend using only names or only indexes)
  2. A Table
