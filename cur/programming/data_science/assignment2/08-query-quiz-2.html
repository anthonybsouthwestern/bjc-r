Which querying blocks could we use (in order from first to last) to best answer the following question about our taxis dataset: What percentage tip did the passenger on the longest ride in the dataset leave?

LIMIT, WHERE, SELECT
Not quite, if we use the LIMIT block first, we would potentially eliminate the row with the longest ride. This is because we do not know where the longest ride is located in the Table, so we cannot accurately LIMIT the Table to assuredly include that row.
ORDER, LIMIT, SELECT
Correct! We can first ORDER the Table by distance in descending order, then LIMIT the Table to just the longest ride (the first row of the ordered Table), and finally SELECT the tip and total columns to be able to answer the question with arithmetic.
WHERE, LIMIT, SELECT
Not quite, although you could use the WHERE block to find the row with the max distance, this would only leave you with one row. Even though this is the row we want to use, the use of the LIMIT block immediately after that is redundant because we do not have any rows we want to eliminate. Always try to make your queries as precise as possible!
DROP, ORDER, LIMIT
Not quite, although you could use these three queries in this order to answer the question, we should not use the DROP block in this case because we are interested in 3 of the 18 columns. When we are interested in a small amount of the columns, we should use SELECT instead of DROP to avoid having to write out each of the several columns we want to DROP.
