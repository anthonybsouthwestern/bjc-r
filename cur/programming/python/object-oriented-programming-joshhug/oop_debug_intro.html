One of the most important skills to learn in programming is how to debug and spot errors in your own code. This lab will help you practice debugging, and reading error messages from the terminal.

To get started, download the lab files from this link.

To get started on this lab:
  • Unzip this file. You can unzip this file through the command line by using the unzip command. 
    $ unzip oop-debugging-lab.zip
$ cd oop-debugging-lab
$ ls
oop_debugging.py      quidditch_game.py

    Debugging the Account Class

    Take a look inside oop_debugging.py. There are several bugs in the Account class. Fix the bugs to satisfy the code in the docstring, shown below. The docstring is the text in triple quotes underneath the class header. It also shows examples of what each function should do. You can put this code into your terminal to test it out!

    /Make sure to read the docstrings for EVERY function so you understand what each function should do!

    
            >>> sophia_account = Account('Sophia')
>>> sophia_account.deposit(1000000)   # depositing my paycheck for the week
1000000
>>> sophia_account.transactions
[('deposit', 1000000)]
>>> sophia_account.withdraw(100)      # buying dinner
999900
>>> sophia_account.transactions
[('deposit', 1000000), ('withdraw', 100)]
