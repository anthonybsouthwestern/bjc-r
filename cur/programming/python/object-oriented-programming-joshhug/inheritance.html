As a sneak preview of future CS courses you might take, let's very briefly discuss the idea of "inheritance". Imagine that we want to create objects of the type MooingBook, which is a book that is just like any other book, except that it can make a mooing sound. A MooingBook should have every attribute and method a book has: a genre, title, author, and publication year. However, it will also have a method called moo that lets it moo. Writing out all that code again would be tedious and reptetitive! Instead we can do this: 


class MooingBook(Book):    
    def moo(self):
        print("On behalf of " + self.author +  ", I moo.")

At first glance, it might seem that our new class just has a moo method.
Well, load up your Python file by entering "python3 -i oop_labv3.py" at your command line, then try the following out in the terminal: 


>>> math_textbook = MooingBook('Mathematics', 'Mathematics Textbook', 'Euclid', 324)

Now, try asking for its language, genre, and author. What do you see? The reason for this is that MooingBook is a subclass of Book and so all the things we defined in Book are automatically defined in MooingBook as well! This is known as inheritance in OOP! 


>>>math_textbook.genre
'Mathematics'
>>>math_textbook.author
'Euclid'
>>>math_textbook.moo()
'On behalf of Euclid, I moo.'

We can create as many subclasses of our main class as we want, and subclasses of those subclasses, and subclasses of those subclasses, and so on and so forth. The key idea to remember however, is that when you look up an attribute or a method of an object, it will start looking at the bottom of the pyramid and work its way up. So, for our example above, when we asked for the genre of our MooingBook, Python actually started by looking in the MooingBook class, and when it did not find any attribute with that name, it went and looked in the superclass Book, finding it there. We won't really use inheritance in our course, but you'll see this idea if you take 61A.
