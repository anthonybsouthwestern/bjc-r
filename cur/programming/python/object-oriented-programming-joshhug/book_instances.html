Now let's think about some aspects that we want to be unique to each book. For example, each book should have its own title, author, genre, and year of publication. Since these characteristics vary from one book object to the next, we refer to them as instance attributes. The first thing to know is that instance attributes take precedence over class attributes. So, if you ask for an object's attribute and it has both a class and an instance attribute by that name, it will return the instance attribute. Coding these attributes is a bit trickier than with class attributes. In order to do so, we have to first talk about how to create individual objects of the class Book and access their attributes.

As a refresher, here is our code so far: 


class Book:
    language = 'English'
    worn_down = False

At this point, in order to create a book object, we would type out something like this: 


>>> example_book = Book()
>>> example_book
<__main__.Book object at 0x000001FAC7F7F320>
>>> example_book.language
'English'

Notice how we were able to ask for the language of our book above. This can be done with all attributes, using dot notation. Here's how dot notation works: on the left side of the period we type the name which our object is bound to (in this case example_book), and on the right side we type the attribute we want. But what if we wanted to change of our instance attributes? How do we assign or reassign them? There are 2 ways to do this.
